What it does

Everything renders inside the browser tab — nothing extra to install.

Auto-matched files

Pick a folder; each video finds its same-named .vtt/.srt subtitles and .json/.txt metadata automatically.

Local, Drive, or web

Play local files, YouTube links, public Drive URLs, private videos chosen through a read-only Drive browser, and direct HTTP(S) media.

Remote buffering

Stream normally or buffer a complete remote file in memory before playback, with progress and safe fallback.

Constant metadata

A persistent overlay (title, date, anything) stays on screen for the whole clip — perfect for context during a lecture or talk.

Synced subtitles

Custom WebVTT renderer styled for readability on a shared screen. Resize and reposition on the fly.

Responsive overlays

Timecode, metadata, and subtitles adapt to the video area while keeping manual size and position controls.

Presentation mode

One key hides all the chrome so the video fills the tab — clean to share.

Local-file privacy

Files you select stay inside your browser. Linked videos connect directly to the source host you chose.

How it works

Name the files with the same base name and drop them in a folder:

my-folder/
├── talk.mp4      ← the video
├── talk.vtt      ← subtitles  (optional, or talk.srt)
└── talk.json     ← metadata overlay  (optional, or talk.txt)

Metadata as JSON renders as key: value lines:

{
  "Lecture": "Linear Regression — Introduction to ML",
  "Topic": "Fitting a line to data",
  "Date": "30 Jun 2026"
}

A plain talk.txt is shown as-is.

Don't want to hand-write JSON? The app has a built-in ✎ New metadata button — type fields, see a live preview, and download a ready-to-use .json (or apply it to the current video instantly).

Then:

  1. Open the app and click Pick folder (or drag a folder onto the page).
  2. Click a video in the sidebar — subtitles and metadata load automatically.
  3. Share the Chrome Tab in Meet/Zoom and tick “Also share tab audio.”
Playback uses the browser's codecs — MP4 (H.264) and WebM are the safe bets. .mkv/.avi may not decode in Chrome.
Linked video: click ＋ Add link for YouTube, a public Google Drive video file, or a direct HTTP(S) media URL. For a private Drive video, use Menu → Open from Google Drive; the app requests read-only Drive access, filters the browser to folders and videos, and cannot change files. Remote links stream with browser preloading, while private Drive files and the optional safety mode use a bounded in-memory buffer. Same-named subtitle and metadata sidecars still attach.

Run locally — just download one file

No install or sign-up. Local files stay in your browser; remote links connect only to their source host. The whole app is the single file app.html.

1. Download app.html

# macOS / Linux
curl -O https://raw.githubusercontent.com/nipunbatra/video-subtitle-overlay/main/app.html

# Windows (PowerShell)
iwr https://raw.githubusercontent.com/nipunbatra/video-subtitle-overlay/main/app.html -OutFile app.html

No terminal? Use Menu → Download raw HTML inside the app, or open the raw file and choose Save Page As… → app.html.

2. Open it in Chrome

macOS    open -a "Google Chrome" app.html
Linux    google-chrome app.html        # or: xdg-open app.html
Windows  start chrome app.html         # or just double-click the file
A Chromium browser (Chrome, Edge, Brave) is recommended — Pick folder and “share tab audio” work best there. Prefer not to download? The hosted app works too and still keeps every file on your machine.

You can now paste a shared Google Drive video file directly in the app. Folder links are not supported; use the individual file's sharing link.
