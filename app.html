Open a video

Pick folder loads local videos and their matching subtitle or metadata files.

＋ Add link opens YouTube, public Drive, or direct video URLs. For a private Drive file, use Menu → Open from Google Drive.

Files stay in your browser. Private Drive access is read-only: the app can browse names and folders and download the video you choose, but never Gmail or Drive changes.

File naming and sidecars
Give related files the same base name:
• talk.mp4 — video
• talk.vtt / talk.srt — subtitles
• talk.json / talk.txt — metadata

For linked videos, the base name is the YouTube/Drive ID or remote filename. New metadata can apply directly to the current video.
More ways to add and share
Drag files, a whole folder, or a supported video URL onto the page. For screen sharing, share the Chrome tab and enable Share tab audio; every overlay appears in the shared view.

Add a video link

Paste a YouTube link, a Google Drive file link, or a direct https://…/video.mp4 URL. Drive files must be shared as Anyone with the link and allow downloads.

Open a private Drive video

Connect a Google Account, browse your Drive folders, and choose a video. The app never requests Gmail access and cannot upload, edit, share, or delete Drive files.

Read-only permission: browse Drive file names and folders, then download the video you choose (drive.readonly). The short-lived access token stays in memory and is never written to local storage.
Site-owner setup
Needed once per deployment—not per user. This public browser identifier is never a client secret or refresh token. A fixed hosted app should place it in BUILTIN_GOOGLE_CONFIG. No API key is needed. See the setup instructions.
Enter the deployment credentials once, then save them.
